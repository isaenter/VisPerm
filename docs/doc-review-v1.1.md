# VisPerm 完整项目文档 V1.1 评审意见

## 总体结论

- 评审结论：**有条件通过（Approve with comments）**。
- 当前文档覆盖了从立项、需求、架构、数据模型到测试与交付的完整链路，结构合理，能够作为项目启动基线。
- 建议在进入开发前补齐「可执行细节」与「可量化标准」，特别是：权限语义精确定义、性能容量基线、安全合规细项、版本治理与灰度策略。

## 文档亮点

1. **范围完整性高**：包含项目边界、MVP、实施计划、验收标准、风险管理，具备“可立项”所需主要章节。
2. **架构分层清晰**：表现层/接口层/应用层/引擎层/基础设施层划分明确，便于团队并行开发。
3. **权限引擎思路正确**：定义了四类边语义及拒绝优先原则，适合企业权限复杂场景。
4. **工程化意识较强**：有缓存、降级、审计、快照、回滚、测试、交付物与协作规范，体现平台化设计。

## 主要问题与改进建议（按优先级）

## P0（上线前必须补齐）

### 1) 权限语义存在歧义，可能导致实现不一致
- `EXTENSION=APPEND`、`INHERITANCE=UNION` 在资源层面语义接近，但在数据范围层面的合并规则没有精确定义。
- 缺少“资源授权结果 + 数据过滤表达式”在多路径汇总时的确定性算法（同优先级冲突时如何裁决）。

**建议补充：**
- 明确分层裁决模型：
  - 资源层：Allow/Deny 集合运算。
  - 数据层：按资源维度合并表达式（AND/OR）及优先级。
- 增加规则冲突决策表（同资源、不同路径、不同边类型、不同优先级）。

### 2) 多租户模型未落库，存在数据串租风险
- 文档提到多租户场景，但核心表（如 `vis_node/vis_edge/vis_snapshot/vis_audit_log`）缺少 `tenant_id` 强约束字段及联合索引策略。

**建议补充：**
- 所有业务表增加 `tenant_id`（必要时含 `env`），关键唯一键改为租户内唯一。
- 增加 Row Level Security（或等价逻辑）策略说明。
- API 层要求租户上下文必传并可审计。

### 3) 安全设计缺少“防越权”细化清单
- 目前仅给出原则，缺少落地机制（如 IDOR、防重放、签名校验、最小权限发布流程）。

**建议补充：**
- 增加接口级安全基线：鉴权、鉴租、鉴资源三层检查。
- 增加高危操作二次确认与审批流（至少预留扩展点）。
- 审计日志增加不可篡改策略（WORM/外部日志仓）。

## P1（建议在迭代1内补齐）

### 4) 性能目标缺少容量前提
- “常规角色计算 <200ms”缺少输入规模定义：节点/边数量、规则复杂度、并发QPS。

**建议补充：**
- 给出基准工况（例如：500节点、1500边、规则深度<=6、QPS=200）。
- 增加压测维度：冷启动、缓存穿透、Redis抖动、版本切换瞬时峰值。

### 5) API 规范缺少幂等性与并发控制
- 写接口未定义幂等键、乐观锁版本号、冲突码约定。

**建议补充：**
- `PUT/POST` 写接口支持 `Idempotency-Key`。
- 图保存使用 `version` + `If-Match`（或等价字段）防并发覆盖。
- 统一错误码补充：并发冲突、版本过期、规则非法、依赖缺失。

### 6) 快照回滚缺少发布治理
- 回滚动作未定义是否全量即时生效、是否支持灰度/租户维度生效。

**建议补充：**
- 增加发布模型：草稿 → 预发布 → 正式。
- 增加灰度策略：按租户/角色/用户组分批生效。
- 增加“回滚后自动失效缓存 + 事件广播”流程。

## P2（优化项）

### 7) 前端体验可进一步提升
- 画布交互较完整，但缺少可观测性辅助（例如路径解释可复制、规则比对可视化）。

**建议补充：**
- 模拟运行页增加“路径分步解释”和“规则差异高亮”。
- 大图场景增加虚拟渲染与分层加载策略说明。

### 8) 测试章节可增加质量门禁
- 目前列了类型，但未定义准入门槛。

**建议补充：**
- 覆盖率门槛（如核心引擎单测 >=85%）。
- 关键用例基线（拒绝优先、环路、回滚、降级）必须全绿才可发布。

## 建议新增的文档章节

1. **术语与语义规范**（RBAC/ABAC、Allow/Deny、Scope、Policy、Effect）
2. **错误码与异常字典**（含可恢复/不可恢复分类）
3. **SLO/SLA 与容量规划**（时延、可用性、恢复时间目标）
4. **发布与变更管理**（审批、灰度、回滚剧本）
5. **合规与数据治理**（日志留存周期、脱敏策略、跨境约束）

## 可直接落地的下一步（建议执行顺序）

1. 冻结一版 `术语+裁决语义` 文档，作为前后端与测试统一依据。
2. 先补数据库多租户字段与索引设计，再进入接口开发。
3. 定义发布/回滚状态机（草稿、已发布、回滚中、已回滚）。
4. 建立 P0 场景自动化用例，接入 CI 门禁。
